## Key Learnings
- **Production DB is named `odoo`** (db 1); fresh DB is `odoo_repair_fresh` (db 2). 140 helpdesk tickets in production.
- **Studio "related stored" fields are silently dropped on export** — module had `x_studio_rug_confirmed` as plain Boolean while production has it as `related='task_id.helpdesk_ticket_id.x_studio_rug_confirmed', store=True`. This is the pattern that propagates RUG state across Ticket → SO → Invoice via `x_studio_sale_id`.
- **Studio "multi" server actions chain via `rel_server_actions` table** — prod action 1980 ("Request RUG Approval") parent has 2 children: 1983 (object_write stub) and 1985 (`next_activity` schedules "Approve RUG Repair" activity for `user_id`).
- **Odoo blocks direct writes to `account.move.line` on posted entries.** The "Update RUG Account" mechanism requires `button_draft()` → write → `action_post()` wrapper.
- **Studio's `journal.default_account_id` was wrong basis** for finding invoice income lines — uses account 21 (400000) but real lines use 1211 (MI001). The correct search is `account_id.user_type_id.internal_group = 'income'`.
- **Field XML IDs auto-generated by Odoo:** `helpdesk_repair_custom.field_<model>__<field>` (double underscore between model and field, single underscore in model name dots become single underscores). Used in `on_change_field_ids` refs.
- **Active = boolean column doesn't exist on `ir_act_server`** in this Odoo 15 schema — but does on `base_automation`. Studio "duplicate" server actions live forever; only their parent automation can be deactivated.

## Solutions & Fixes
- **Production DB list:** `sudo -u postgres psql -lA` (peer auth as postgres works; `psql -U odoo` doesn't).
- **RUG field inventory query:**
  ```sql
  SELECT model, name, ttype, COALESCE(related,'') as related, store::text
  FROM ir_model_fields WHERE name ILIKE '%rug%' OR field_description ILIKE '%rug%'
  ORDER BY model, name;
  ```
- **RUG server-action code inventory:**
  ```sql
  SELECT s.id, s.name, im.model, s.state
  FROM ir_act_server s LEFT JOIN ir_model im ON im.id=s.model_id
  WHERE s.name ILIKE '%rug%' OR s.code ILIKE '%rug%' OR s.code ILIKE '%x_studio_rug%';
  ```
- **Upgrade command (db 2):**
  ```bash
  sudo -u odoo15 /opt/odoo15/myodoo15-venv/bin/python3 /opt/odoo15/odoo/odoo-bin \
    -u helpdesk_repair_custom -d odoo_repair_fresh \
    --addons-path=/opt/odoo15/odoo/addons,/opt/odoo15/custom-addons \
    --db_user=odoo15 --stop-after-init --logfile=/tmp/odoo_upgrade.log
  ```
- **Service restart:** `sudo systemctl restart odoo15` (config at `/etc/odoo15.conf`).
- **Verifying parent/child action chains:** `SELECT * FROM rel_server_actions WHERE server_id IN (...)` — columns are `server_id` (parent) + `action_id` (child).
